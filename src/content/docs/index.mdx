## Why this site exists

CGIAR Climate Action publishes climate-rationale tools that surface future-climate projections to people writing Green Climate Fund concept notes, Adaptation Fund proposals, NAP submissions, and research-for-development scenario briefs. Behind every chart in those tools sit dozens of methodology choices — which climate models, which scenarios, which downscaling product, which regional tuning — and those choices need to be **defensible to the partners who will cite them**.

The first body of reference material here covers [climate data for African adaptation](/wikis/aaa-atlas/) — what a projection is and isn't, which baselines to use when, how CMIP6 ensembles are chosen and regionally tuned, what the dataset landscape looks like, and the conventions CGIAR centres should harmonise around. Implementation experience comes from operational tools including the [AAA Adaptation Atlas](https://adaptationatlas.cgiar.org/)'s *Build a Climate Rationale* notebook, but the answers — *which ensembles defensibly work for each African region, what the baseline asymmetry between observational and projection products means, what to do about the East African paradox* — apply to any African climate-rationale work using CMIP6. Further wikis in other domains will follow.

## About this site

Authoring happens here in plain Markdown (with optional MDX for richer components); the site is built with [Astro Starlight](https://starlight.astro.build/) and deployed to GitHub Pages on every push to `main`. Source of truth varies per wiki — some are drafted in their originating CGIAR programme's repository and synced here; others are authored directly. Each page's frontmatter makes the source explicit.
